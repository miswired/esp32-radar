# Change: Add Comprehensive Documentation

## Why
The project needs professional, well-organized documentation to support open source adoption. Users of all skill levels need clear guidance for hardware assembly, configuration, and integration. Contributors need understanding of the codebase architecture and design decisions.

## What Changes
- Add GPL v3 license to the repository
- Add license notice to web interface (About page)
- Create GitHub Wiki documentation following Diataxis framework:
  - **Tutorials**: Step-by-step learning experiences for beginners
  - **How-to Guides**: Goal-oriented instructions for specific tasks
  - **Reference**: Technical specifications and API documentation
  - **Explanation**: Background context and design rationale
- Add contributor documentation (CONTRIBUTING.md, CODE_OF_CONDUCT.md)
- Add project metadata (CHANGELOG.md, hardware BOM, wiring diagrams)

## Impact
- Affected specs: web-interface (new About page)
- Affected code: All stage directories (license headers), web UI (About page)
- New files: LICENSE, CONTRIBUTING.md, CODE_OF_CONDUCT.md, CHANGELOG.md, docs/
- GitHub Wiki: New documentation pages
